1. Introduction
1.1 Purpose
The purpose of this document is to describe the requirements for the Hotel Room Booking System, which allows users to search, view, and book hotel rooms online. 
1. Functional Requirements (FR)
Functional requirements describe what the system should do.
FR1: User Registration
Users should be able to create an account.
System should store user details (name, email, password).
FR2: User Login
Registered users should be able to log in using email and password.
System should validate user credentials.
FR3: Search Hotels
Users should be able to search hotels by:
Location
Check-in date
Check-out date
Number of guests

Images and reviews
Amenities
Users should be able to sort results by price or rating.
Users should be able to select a room.
Users should enter guest details.
System should generate a booking confirmation.

2. Non-Functional Requirements (NFR)
Non-functional requirements describe how the system performs.
NFR1: Performance
System should display search results within 3 seconds.
System should support multiple users simultaneously.
NFR2: Security
User passwords should be encrypted.
Payment transactions should be secure.
System should use authentication and authorization.
NFR3: Availability
System should be available 24/7.
System uptime should be 99% or higher.
NFR4: Usability
System interface should be simple and easy to use.
Users should be able to complete booking in few steps.
NFR5: Reliability
System should store data accurately without loss.
Backup should be performed regularly.
NFR6: Scalability
System should handle increasing numbers of users and bookings.
